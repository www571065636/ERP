# ERP系统数据库设计文档

## 设计规范

### 公共字段约定

所有业务表均继承以下基础字段（通过 Django `BaseModel` 实现）：

```python
class BaseModel(models.Model):
    created_by   = models.BigIntegerField(verbose_name='创建人ID')
    created_at   = models.DateTimeField(auto_now_add=True, verbose_name='创建时间')
    updated_by   = models.BigIntegerField(null=True, verbose_name='修改人ID')
    updated_at   = models.DateTimeField(auto_now=True, verbose_name='修改时间')
    is_deleted   = models.BooleanField(default=False, verbose_name='逻辑删除')
    tenant_id    = models.BigIntegerField(default=0, verbose_name='租户ID（预留）')

    class Meta:
        abstract = True
```

### 命名规范

- 表名：`模块前缀_业务名`，全小写下划线，如 `pur_order`
- 字段名：全小写下划线，如 `order_no`
- 主键：统一使用 `id`（BIGINT AUTO_INCREMENT）
- 外键字段：`关联表名_id`，如 `supplier_id`
- 状态字段：`status`，TINYINT，注释说明各值含义
- 金额字段：DECIMAL(15,2)，数量字段：DECIMAL(15,4)

## 一、用户管理模块（system）

### 1.1 sys_user — 系统用户表

```sql
CREATE TABLE sys_user (
  id               BIGINT          NOT NULL AUTO_INCREMENT COMMENT '用户ID',
  username         VARCHAR(64)     NOT NULL COMMENT '登录账号',
  password         VARCHAR(128)    NOT NULL COMMENT '密码（bcrypt加密）',
  real_name        VARCHAR(64)     NOT NULL COMMENT '真实姓名',
  avatar           VARCHAR(255)    COMMENT '头像URL',
  email            VARCHAR(128)    COMMENT '邮箱',
  mobile           VARCHAR(20)     COMMENT '手机号',
  dept_id          BIGINT          COMMENT '所属部门ID',
  status           TINYINT(1)      NOT NULL DEFAULT 1 COMMENT '状态 1启用 0禁用',
  last_login_ip    VARCHAR(64)     COMMENT '最后登录IP',
  last_login_at    DATETIME        COMMENT '最后登录时间',
  pwd_updated_at   DATETIME        COMMENT '密码最后修改时间',
  -- 公共字段
  created_by       BIGINT          NOT NULL COMMENT '创建人ID',
  created_at       DATETIME        NOT NULL DEFAULT CURRENT_TIMESTAMP,
  updated_by       BIGINT          COMMENT '修改人ID',
  updated_at       DATETIME        ON UPDATE CURRENT_TIMESTAMP,
  is_deleted       TINYINT(1)      NOT NULL DEFAULT 0,
  tenant_id        BIGINT          NOT NULL DEFAULT 0,
  PRIMARY KEY (id),
  UNIQUE KEY uk_username (username, tenant_id),
  KEY idx_dept_id (dept_id),
  KEY idx_mobile (mobile)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='系统用户表';
```

### 1.3 sys_permission — 权限菜单表

```sql

## 九、索引策略总结

| 场景 | 索引类型 | 说明 |
|------|---------|------|
| 单号查询 | UNIQUE | order_no、receipt_no 等业务单号 |
| 外键关联 | KEY | supplier_id、customer_id、product_id 等 |
| 日期范围查询 | KEY | order_date、created_at 等时间字段 |
| 状态过滤 | KEY | status 字段（配合其他条件使用） |
| 库存唯一约束 | UNIQUE | (warehouse_id, product_id, sku_id, tenant_id) |
| 多租户隔离 | 复合UNIQUE | 业务唯一键均包含 tenant_id |
```
