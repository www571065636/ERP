# ERP系统数据库设计文档

## 文档说明

本文档以**当前 Django ORM 模型与现有业务字段**为基准，整理数据库结构现状，方便开发与联调。

> 说明：部分历史设计文档中的字段（如 `tenant_id`、`exchange_rate`、`discount_amount`、`volume` 等）属于目标设计或曾规划字段，当前 ORM 中并未全部实现，本文不再将其写成现状。

---

## 设计规范

### 公共字段约定

当前项目中，绝大多数业务主表继承以下基础字段（定义于 `backend/common/models.py`）：

```python
class BaseModel(models.Model):
    created_by = models.BigIntegerField(default=0, verbose_name="创建人")
    created_at = models.DateTimeField(auto_now_add=True, verbose_name="创建时间")
    updated_by = models.BigIntegerField(default=0, verbose_name="更新人")
    updated_at = models.DateTimeField(auto_now=True, verbose_name="更新时间")
    is_deleted = models.BooleanField(default=False, verbose_name="是否删除")

    class Meta:
        abstract = True
```

说明：
- 当前 `BaseModel` **不包含** `tenant_id`
- 行明细表通常不继承 `BaseModel`
- 删除策略以逻辑删除为主，但个别模块仍有物理删除实现，需以后端代码为准

### 数据库现状

当前默认数据库配置位于：
- `backend/config/settings.py`

当前默认使用：
- SQLite（开发环境）

如需切换 MySQL，应以后续配置改造为准，而非本文当前现状。

## 八、当前未落地但常被误解的设计项

以下字段或结构曾在旧设计中出现，但当前 ORM 不应视为现状：

- `tenant_id`
- `pwd_updated_at`
- `exchange_rate`
- `discount_amount`
- `volume`
- `capacity`
- 基于 `tenant_id` 的联合唯一索引

如果后续需要恢复这些能力，应以代码改造与迁移脚本为准。
