# ERP系统设计文档

## 1. 系统概述

本系统是一套面向中小企业的通用ERP（企业资源计划）管理平台，覆盖用户权限、产品、采购、销售、库存、财务、人力资源七大核心业务域，实现企业核心业务流程的数字化管理与数据互通。

**核心目标：**
- 统一数据源，消除各部门信息孤岛
- 标准化业务流程，支持多级审批
- 实时库存与财务联动，降低运营风险

## 5. RBAC 权限模型

### 5.1 模型关系

```
User（用户）
  └── ManyToMany ──► Role（角色）
                        └── ManyToMany ──► Permission（权限）
                                              ├── 菜单权限（前端路由控制）
                                              ├── 按钮权限（前端元素控制）
                                              └── API权限（后端接口控制）
```

### 5.2 权限标识符规范

格式：`模块:资源:操作`

```
system:user:list        # 用户列表查看
system:user:create      # 新增用户
system:user:update      # 修改用户
system:user:delete      # 删除用户
purchase:order:approve  # 采购订单审批
finance:voucher:post    # 财务凭证过账
inventory:stock:adjust  # 库存调整
```

### 5.3 数据权限维度

通过自定义 DRF Permission 类 + QuerySet 过滤实现行级数据隔离：

| 数据范围 | 说明 |
|---------|------|
| `ALL` | 全部数据 |
| `DEPT` | 本部门数据 |
| `DEPT_AND_CHILD` | 本部门及下级部门数据 |
| `SELF` | 仅本人创建的数据 |
| `CUSTOM` | 自定义指定部门数据 |

### 5.4 Django 实现方式

```python
# 自定义权限后端
class RBACPermissionBackend:
    def has_permission(self, request, view):
        required_perm = getattr(view, 'required_permission', None)
        if not required_perm:
            return True
        return request.user.has_erp_permission(required_perm)

# ViewSet 中声明所需权限
class PurchaseOrderViewSet(ModelViewSet):
    required_permission = 'purchase:order:list'
```

## 6. 核心设计模式

### 6.1 Django 信号解耦跨模块通知

采购收货确认后，通过 Django Signal 触发库存入库和财务应付生成，避免直接跨 App 调用：

```python
# purchase/signals.py
receipt_confirmed = Signal()  # 收货确认信号

# inventory/receivers.py
@receiver(receipt_confirmed)
def create_stock_in(sender, receipt, **kwargs):
    # 生成库存入库流水
    ...

# finance/receivers.py
@receiver(receipt_confirmed)
def create_payable(sender, receipt, **kwargs):
    # 生成应付账款
    ...
```

### 6.2 select_for_update 防并发超卖

库存扣减使用数据库行锁，防止并发场景下库存为负：

```python
with transaction.atomic():
    stock = InvStock.objects.select_for_update().get(
        warehouse_id=warehouse_id,
        product_id=product_id
    )
    if stock.qty_available < qty:
        raise BusinessException("库存不足")
    stock.qty_available -= qty
    stock.save(update_fields=['qty_available'])
```

### 6.3 Celery 异步任务

耗时操作异步处理，避免阻塞 HTTP 请求：
- 月度薪资批量核算
- 财务报表生成（Excel导出）
- 库存预警邮件通知
- 采购/销售订单审批提醒

### 6.4 幂等性保障

所有写操作通过 Redis SETNX 防重复提交：

```python
class IdempotentMixin:
    def create(self, request, *args, **kwargs):
        idempotent_key = request.headers.get('X-Idempotent-Key')
        if idempotent_key:
            cache_key = f"idempotent:{idempotent_key}"
            if not cache.add(cache_key, 1, timeout=300):
                raise ValidationError("重复请求")
        return super().create(request, *args, **kwargs)
```

## 9. 核心业务流程

### 采购流程
```
创建采购订单 → 提交审批 → 审批通过 → 供应商发货
    → 创建收货单 → 确认收货 → [触发信号]
        ├── 库存入库（inv_transaction）
        └── 生成应付账款（fin_payable）
```

### 销售流程
```
创建销售订单 → 提交审批 → 审批通过 → 创建发货单
    → 确认出库 → [触发信号]
        ├── 库存出库（inv_transaction）
        └── 生成应收账款（fin_receivable）
```

### 库存流水类型

| 类型代码 | 说明 | 数量方向 |
|---------|------|---------|
| `PURCHASE_IN` | 采购入库 | + |
| `SALE_OUT` | 销售出库 | - |
| `TRANSFER_IN` | 调拨入库 | + |
| `TRANSFER_OUT` | 调拨出库 | - |
| `ADJUST_IN` | 盘盈调整 | + |
| `ADJUST_OUT` | 盘亏调整 | - |
| `RETURN_IN` | 销售退货入库 | + |
| `RETURN_OUT` | 采购退货出库 | - |
