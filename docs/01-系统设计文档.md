# ERP系统设计文档

## 1. 文档说明

本文档描述**当前代码库已经实现的架构与能力边界**，用于帮助开发、联调与测试快速理解系统现状。

> 说明：仓库中仍保留部分更完整的目标架构设想，例如 MySQL、Redis、Celery、MinIO、采购收货自动入库、销售发货自动出库、库存盘点、数据权限等；这些能力**当前版本尚未完整落地**，不能视为现状。

---

## 2. 系统概述

本系统是一套面向中小企业的 ERP 原型系统，覆盖以下七个业务域：

- 用户权限
- 产品管理
- 采购管理
- 销售管理
- 库存管理
- 财务管理
- 人力资源

当前版本已完成主要模块的基础数据管理、部分单据状态流转与前后端页面联通，适合继续在此基础上完善业务闭环和权限控制。

---

## 3. 当前整体架构

### 3.1 当前运行架构

```text
Vue 3 + Element Plus + Pinia
            │
            │ HTTP
            ▼
Django + Django REST Framework
            │
            ▼
      SQLite（开发库）
```

### 3.2 当前代码中的实际技术栈

| 技术 | 当前情况 | 说明 |
|------|---------|------|
| Python | 已使用 | 后端运行环境 |
| Django | 已使用 | Web 框架 |
| Django REST Framework | 已使用 | REST API |
| SimpleJWT | 已使用 | JWT 登录认证 |
| django-filter | 已使用 | 列表过滤 |
| Vue 3 | 已使用 | 前端框架 |
| Element Plus | 已使用 | 前端 UI 组件 |
| Pinia | 已使用 | 前端状态管理 |
| SQLite | 已使用 | 当前 `backend/config/settings.py` 中的默认数据库 |
| MySQL | 未在当前配置中启用 | 属于规划/生产化方向 |
| Redis | 未落地 | 当前未配置缓存、分布式锁、队列 |
| Celery | 未落地 | 当前未配置异步任务 |
| MinIO | 未落地 | 当前未配置文件对象存储 |

相关文件：
- `backend/config/settings.py`
- `frontend/src/main.js`

## 4. 模块划分与现状

| Django App | 路由前缀 | 当前已实现 |
|-----------|---------|-----------|
| `system` | `/api/v1/system` | 用户、角色、权限 CRUD，权限树，登录态相关接口 |
| `product` | `/api/v1/products` | 产品、分类、计量单位、SKU 基础管理 |
| `purchase` | `/api/v1/purchase` | 供应商、采购订单、提交审批 |
| `sales` | `/api/v1/sales` | 客户、销售订单、提交审批 |
| `inventory` | `/api/v1/inventory` | 仓库、库存台账查询、库存流水查询 |
| `finance` | `/api/v1/finance` | 会计科目、财务凭证、应收、应付 |
| `hr` | `/api/v1/hr` | 员工、考勤、薪资单、批量生成薪资、批量发放 |

根路由文件：
- `backend/config/urls.py`

## 6. 当前 RBAC 现状

当前代码已经具备：
- 用户、角色、权限三层数据模型
- 登录后返回权限码列表
- 前端 `auth.hasPermission()` 能力预留

但当前版本**尚未完整实现真正生效的 RBAC**，具体表现为：
- 后端目前主要依赖 `IsAuthenticated`，还未完成基于 `perm_code` 的统一接口级拦截
- 前端菜单、路由、按钮大多仍是静态展示，未全面按权限显隐
- `data_scope` 字段已存在，但行级数据权限尚未落地

相关文件：
- `backend/config/settings.py`
- `backend/system/models.py`
- `backend/system/views.py`
- `frontend/src/stores/auth.js`
- `frontend/src/router/index.js`
- `frontend/src/layouts/MainLayout.vue`

## 10. 当前业务流程边界

### 采购流程（当前已实现到审批）

```text
创建采购订单 → 提交审批 → 审批通过/驳回
```

### 销售流程（当前已实现到审批）

```text
创建销售订单 → 提交审批 → 审批通过/驳回
```

### 财务凭证流程（当前已实现）

```text
新增凭证 → 审核 → 过账
```

### 薪资流程（当前已实现）

```text
生成薪资单 → 审核 → 批量发放
```
